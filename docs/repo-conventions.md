# Repository Conventions

## Overview

SpecStat uses an additive file convention. It **never modifies your OpenSpec files** — it only adds `visualize.json` files alongside them.

## Required: `openspec/index.json`

The only hard convention is a file at `openspec/index.json` at your repo root. This is auto-generated by the `openspec-init` action and kept up to date by `openspec-sync`.

SpecStat reads this file first when loading a repo.

## Folder structure

```
openspec/
  index.json                  ← root manifest (auto-generated)
  visualize.json              ← root section metadata
  specs/
    visualize.json            ← folder metadata
    AUTH-001/
      AUTH-001.md             ← your spec (never touched)
      visualize.json          ← item metadata (auto-generated)
    AUTH-002/
      AUTH-002.md
      visualize.json
  designs/
    visualize.json
    DESIGN-001/
      DESIGN-001.md
      visualize.json
  tasks/
    visualize.json
  decisions/
    visualize.json
  archive/
    specs/
      visualize.json
      AUTH-OLD-001/
        AUTH-OLD-001.md
        visualize.json
```

## Mixed repos (spec + code)

The `openspec/` directory can live anywhere in your repo alongside code. SpecStat always looks for `openspec/index.json` at the repo root.

## Naming conventions

- Item folders: use the item ID as the folder name (e.g., `AUTH-001/`)
- Spec files: same as folder (e.g., `AUTH-001.md`)
- `visualize.json` must be lowercase, no variation

## Archive

Items marked `archived: true` in their `visualize.json` are excluded from the main board and graph views. They appear in the dedicated Archive view.

To archive an item, either:
1. Edit its `visualize.json` to set `"archived": true`
2. Add `<!-- @visualizer:trigger archive -->` to the spec file — the next sync will process it
