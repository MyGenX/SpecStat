# visualize.json Schema Reference

## Item-level `visualize.json`

Every spec item directory must contain a `visualize.json` file. Required fields are marked with *.

```json
{
  "type": "item",           // * always "item"
  "id": "AUTH-001",         // * unique item identifier
  "title": "User login flow", // * human-readable title
  "spec_type": "spec",      // * one of: spec | impl | task | design | proposal | decision | component | domain
  "status": "approved",     // * one of: draft | in-review | approved | implemented | deprecated | archived
  "priority": "high",       // null | high | medium | low
  "owner": "platform-team", // * team or user responsible
  "contributors": ["alice", "bob"],
  "version": "1.2.0",
  "tags": ["auth", "security"],
  "archived": false,
  "spec_file": "AUTH-001.md", // * filename of the spec (relative to this folder)
  "created_at": "2025-06-01", // * ISO date
  "last_updated": "2026-01-15",
  "last_commit": "9f3ab21",
  "card": {
    "color": "#4F46E5",
    "icon": "lock",
    "cover_image": null,
    "summary": "Defines the complete login flow"
  },
  "relations": {
    "relates_to": ["AUTH-002"],
    "depends_on": ["COMP-AUTH-SERVICE"],
    "implements": ["PROP-007"],
    "linked_tasks": ["TASK-042"],
    "linked_designs": ["DESIGN-001"],
    "supersedes": [],
    "superseded_by": null
  },
  "github": {
    "labels": ["critical", "security"],
    "linked_pr": 52,
    "linked_issue": 103,
    "milestone": "v2.0"
  },
  "board": {
    "column": "approved",
    "position": 2,
    "swimlane": "authentication"
  },
  "triggers": {
    "on_status_change": "openspec-notify",
    "on_approve": "openspec-baseline"
  }
}
```

### Valid `spec_type` values

| Value | Meaning |
|---|---|
| `spec` | Functional or non-functional requirement |
| `impl` | Implementation spec |
| `task` | Actionable task |
| `design` | Design document |
| `proposal` | Change proposal |
| `decision` | Architecture decision record |
| `component` | Component definition |
| `domain` | Domain boundary |

### Valid `status` values

| Value | Meaning |
|---|---|
| `draft` | Work in progress |
| `in-review` | Under review |
| `approved` | Approved and baselined |
| `implemented` | Implemented in code |
| `deprecated` | No longer valid |
| `archived` | Archived, hidden from main views |

```json
  "default_view": "board",
  "default_group_by": "status",
  "owners": ["platform-team"],
  "tags": ["core", "requirements"],
  "item_count": 12
}
```

## `index.json`

Auto-generated by `openspec-init` and kept current by `openspec-sync`. Do not edit manually.

```json
{
  "meta": {
    "repo": "org/repo-name",
    "repo_type": "mixed",
    "generated_at": "2026-01-15T10:30:00Z",
    "openspec_version": "1.0.0",
    "visualizer_version": "1.0.0",
    "root": "openspec"
  },
  "folders": [ ... ],
  "items": [ ... ]
}
```
